- ##### OOP의 4대 원칙
	- 어디가 객체고 어디가 객체가 아닌가?
		- 폴더에서 리소스는 객체가 아니고, 자바 폴더 안에 있는 파이프라인이 전부 객체. 리소스는 기능이 없기 때문
		- 리소스를 자바로 올리자마자 객체로 만들어야지

- ##### model.addAtribute(" ~)
	- property 와 attribute 차이점 (둘다 속성의 뜻을 갖는데)
	- 속성을 부를 때 html에서는 attribute라고 하고, 자바(oop)에서는 프로펄티라고 한다

|           | attribute    | property            |
| --------- | ------------ | ------------------- |
| 속성을 부르는 곳 | html         | java                |
| " "       | 있음           | 없음                  |
|           | 그냥 넣으면 돼(빠름) | get, set 존재해야 함(느림) |
|           | 변하지 않음       | 변함                  |
- 모든 html은 attribute로 부르고 엘리멘트, 복수형?, 자료구조다. 기능은 없다. 
- < > : 자료구조 
	- html에서 꺽세와 사진에서의 꺽세의 차이: 리스트는 선형 1차원, 유저디티오는 1차원(스키마로 담았으니까). 그래서 전체는 2차원
	-  array는 배열이야. 유져디티오는 타입.		 ![[Pasted image 20251023101054.png]]
	- 참고로 맵은 다차원
	- 왼쪽 보라색 메신저가 attribute, 오른쪽 파란색 메신저를 property

- ##### 괄호 종류

| 종류                     | 의미                               |
| ---------------------- | -------------------------------- |
| ( )                    | 함수, 조건식에서 사용                     |
| { }                    | 블록이라 함: 문, 메소드, 클래스가 가짐          |
| [    ]                 | 배열이라 함, 리스트를 가짐, 1차원에 해당, 차원을 고정 |
| [    ]  [    ]  [    ] | 3차원                              |
| 0차원은 대괄호를 안 쳐          |                                  |

- ##### CRUD

| **약자** | **동작 (한글)** | **데이터베이스 작업 (SQL)** | **비유 (노트)**           |
| ------ | ----------- | ------------------- | --------------------- |
| C      | Create (생성) | `INSERT`            | 새 노트를 작성하고 저장합니다.     |
| R      | Read (조회)   | `SELECT`            | 저장된 노트를 펼쳐서 읽어봅니다.    |
| U      | Update (수정) | `UPDATE`            | 노트의 내용을 고쳐서 다시 저장합니다. |
| D      | Delete (삭제) | `DELETE`            |                       |
|        |             |                     |                       |
