- 비즈니스 로직과 API 구현 클래스
	- API 구현 클래스: 요청을 받고 응답을 내보내는 최전방 역할을 합니다. (보통 Spring의 Controller가 담당)
    - 비즈니스 로직 (Service): 실제로 데이터를 가공하고 처리하는 핵심 로직을 담고 있는 부분입니다.

- 도커
    - 도커: 소프트웨어(애플리케이션, DB 등)를 컨테이너(Container)라는 표준화된 상자에 담아, 어떤 환경에서든 동일하게 실행되도록 보장하는 기술입니다.
    - 비유: 컨테이너 선박 🚢처럼, 안에 무엇(파이썬, 자바, DB)이 들어 있든 규격화된 상자에 담아 전 세계(모든 OS) 어디로든 쉽게 싣고 내릴 수 있게 해줍니다.
        - PaaS (Platform as a Service): 도커는 개발 환경(플랫폼)을 빌려주는 서비스(PaaS)의 형태로 출시되었으며, 사용자는 이 위에 자신의 서비스(SaaS)를 개발합니다.
        - MSA (Microservices Architecture) 기반: 작은 기능 단위로 나누어 개발된 서비스들(마이크로서비스)을 도커 컨테이너에 담아 독립적으로 운영할 수 있게 합니다.
        ![[Pasted image 20251027083154.png]]
    - 도커의 구성요소
	    - 도커 클라이언트 (Client): 사용자가 명령(`docker run`, `docker pull` 등)을 입력하는 터미널(CLI)입니다.
	    - 도커 데몬 (Docker Daemon, 도커D): 백그라운드에서 실행되며 클라이언트의 명령을 받아 컨테이너를 만들고, 실행하고, 관리하는 실제 일꾼입니다.
- 개발 환경의 통일: WSL과 우분투 🐧
    - 리눅스(Linux): 도커는 원래 리눅스 OS의 격리 기능을 사용하기 때문에 리눅스 환경에서 가장 잘 작동합니다. 우분투(Ubuntu)는 리눅스 종류 중 하나이며, 한국의 전자정부 표준 운영체제로도 쓰입니다.
    - WSL (Windows Subsystem for Linux): 윈도우 환경에서 리눅스를 사용하기 위해 개발된 기능입니다. 개발자는 별도의 가상 머신 없이 윈도우 내에서 강력한 리눅스 명령줄 환경(터미널)을 사용할 수 있게 됩니다.
    - `wsl -l -v` 명령어는 현재 윈도우에 설치된 WSL 리눅스 버전 목록과 상태를 보여줍니다.
- 프로세스 개념 🔄
    - 프로세스 (Process): 컴퓨터에서 실행되고 있는 하나의 프로그램을 의미합니다. (필기에서는 '파이프라인'으로 비유됨)
    - 싱글 프로세스: 한 번에 하나의 파이프라인(프로그램)만 실행하는 것.
    - 멀티 프로세스: 하나의 운영체제(OS) 내에서 여러 개의 파이프라인(프로그램)을 동시에 실행하는 것. (컴퓨터가 여러 작업을 동시에 처리하는 능력)

- 터미널: git bash에서 쓰는 것. 리눅스거임. 그래서 터미널이라고 부름
- 파워셸: 나가는 경계선 / 나가면 리눅스(터미널에 있다고 함)
- 윈도우 안 cmd(알맹이-땅) / 실제로 움직이는 건 도커(객체)
- 리눅스랑 윈도우는 땅, 껍데기에 실어 나가니까 파워셸
- 도커세계관에서 자바의 타입은 이미지, 요소
- 도커세계관에서 있을 때는 도커이미지, 메모리로 오면 컨테이너
    - 클래스 인스턴스 = new 클래스();
    - 이미지 컨테이너 = new 이미지();
