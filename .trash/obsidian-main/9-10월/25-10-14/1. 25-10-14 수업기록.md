- Repository: 코드저장소
- 웹 개발의 기본: 모델을 만든 뒤에 뷰를 만들어서 컨트롤러로 연결. 모델을 먼저 만드는 이유는 컴퓨터가 읽는 순서가 모델부터 읽기 때문.
- 폴더는 객체로 취급하지 않으므로 읽을 수 없음
- 컴퓨터는 인덱스를 강제로 먼저 읽음
- form 태그만 그 안에 있는 데이터를 컨트롤러에 보낼 수 있음.
- 보여지는 것 = key, 진짜 속에 잇는 것: value
- 언어는 자연어와 기계어가 있음. 더블쿼터 ” ”로 묶여 있으면 자연어라고 약속함.
- ? : 쿼리 / & : 엔퍼센트
- id는 자바스크립트(로그인때 쓰는 id가 아님)로 보내고 name은 자바로 보냄.
- 공학에서 자연어는 string 이라고 함. string은 charactor의 집합임.
- 아무 값 없어도 스트링은 null 값임
- 쿼리 스트링: [ ?email=a&password=b ]
    - 여기에서 html이 자바한테 쿼리스트링을 준다 라고 표현함
- 프렉탈: 최소단위의 구조가 전체 구조랑 같다
- mapping: 보낸 값을 찾는 과정
- classpath로 정해진 길에 request와 response를 함
    - request: html에서 자바로 가는 길
    - response: 자바에서 html로 가는 길
- 속성(property) = 변수와 상수
    - 변수: 어떤 값을 저장하는 주소 = 위치값 = 움직이는 값(사람) = adress
    - 상수: 주소에서 움직이지 않는 값 = 건물 = 키가 됨
- 객체란 데이터와 메소드의 집합
    - 객체 = 속성 + 기능
    - 개체 = 객체 - 기능 (속성만 있으면 개체, 속성=개체)
    - 함수 = 객체 - 속성 (기능만 있으면 함수, 기능=함수)
    - 모델(객체) = 도메인(속성)+서비스(행위)
- 부모-자식 관계 : 종속관계, 동등한 관계: 시블링
- DDD 구조로 폴더링 변경: Domian - Driven Design 은 도메인을 중심으로 모델링하고 구조를 만드는 방법론
