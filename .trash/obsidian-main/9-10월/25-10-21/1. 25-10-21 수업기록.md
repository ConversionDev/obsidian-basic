- 컴퓨터 작동의 기본 원리 : 컴퓨터가 프로그램을 실행할 때 핵심적인 역할을 하는 세 가지 요소와 데이터의 상태를 설명합니다.
    
- ##### 하드웨어 구성 요소

| 요소            | 풀네임 및 역할                                                                                            |
| ------------- | --------------------------------------------------------------------------------------------------- |
| CPU           | Central Processing Unit (중앙 처리 장치): 모든 연산과 제어를 담당하는 컴퓨터의 '두뇌'입니다. 필기에서 말하는 'Process(작업 처리)'의 주체입니다. |
| 메모리 (RAM)     | 기억, 연산: 현재 실행 중인 데이터와 코드를 임시로 저장하는 공간입니다. loading은 이 메모리에 데이터를 불러오는 행위를 말합니다.                       |
| 디스크 (HDD/SSD) | 저장: 데이터를 영구적으로 저장하는 공간입니다.                                                                          |

- 메모리와 디스크의 차이
    
    - 메모리 (RAM): 휘발성 (전원을 끄면 데이터 소멸). 현재 CPU가 빠르게 접근하여 작업하는 공간.
    - 디스크 (저장소): 비휘발성 (전원을 꺼도 데이터 보존). 영구적으로 데이터를 저장하는 공간.
- 입출력 (I/O)
    
    - 입력 (Input): 데이터를 받아들이는 것 (`System.in` $\rightarrow$ 키보드 입력 등).
    - 출력 (Output): 데이터를 내보내는 것 (`System.out` $\rightarrow$ 화면 출력 등).
- 코드의 구성 요소 및 실행 흐름 제어 📝 프로그램의 최소 단위와 로직의 흐름을 결정하는 문법입니다.
    
    - 식 (Expression): 값이 도출되는 코드 조각입니다. (`3 + 5`, `a * b`, `express` 식이 빠르다(express)는 의미가 붙은 이유는 값으로 바로 처리되기 때문)
    - 문 (Statement): 실행되어야 할 하나의 명령 단위입니다. 끝에 보통 세미콜론(`;`)이 붙습니다.
    - 알고리즘: 식과 문을 조합하여 특정 문제를 해결하는 절차 또는 방법입니다.
    - 함수 (Function) / 메서드 (Method):
        - 함수: 알고리즘에 이름을 붙인 것. (클래스 밖에 있을 때, 자바는 대부분 메서드로 존재)
        - 메서드: 클래스 안에 정의된 함수. 자바는 객체 지향 언어이므로 대부분의 로직은 메서드 형태로 존재합니다.
    - 호출 (Call): 함수나 메서드를 실행하도록 명령하는 행위입니다.

- ##### 제어문 (Control, 분기 및 반복) CPU의 제어 기능을 구현하는 문법입니다

| 제어 형태       | 키워드        | 설명                                                        |
| ----------- | ---------- | --------------------------------------------------------- |
| 분기 (Branch) | if, switch | 조건에 따라 어느 길로 갈지 결정합니다. (if는 조건,<br>switch는 등호(=) 비교에 사용됨) |
| 반복 (Loop)   | for, while | 코드를 반복 실행합니다.                                             |

- 반복문과 제어권 (Stop의 주도권)
    
    - **`for` (횟수 제어):** **프로그램**이 몇 번 반복할지 주도권을 가집니다. (예: 정해진 횟수(3곡)를 채워야 끝남). 주로 배열처럼 **횟수를 알고 있을 때** 사용
    - **`while` (조건 제어):** **사용자**가 멈추고 싶을 때까지 주도권을 가집니다. (예: 게임에서 '계속 하시겠습니까?'에 '아니오'를 입력할 때까지) **무한 루프**는 주로 `while`문에서 발생합니다.
- 자료 구조 심화 및 현대 개발 기술
    - 배열의 확장 (반복)
        - 배열(Array): 문자들이 순서대로 나열된 선형 구조입니다.
            - 2차원: 배열 안의 요소들이 반복되는 구조를 의미합니다. (예: 행렬 `loop counter`를 사용하여 2중 반복문으로 접근)
            - 확장 for문 (`for-each`): 배열이나 리스트의 전체 요소를 처음부터 끝까지 순회할 때 간편하게 사용하는 반복문입니다.
    - POJO와 Spring Bean
        - 현대 자바 프레임워크인 Spring에서 사용되는 중요한 개념입니다.
            - POJO (Plain Old Java Object): 순수한 자바 객체를 의미합니다. 특정 프레임워크 기술에 종속되지 않고, 자바 문법(클래스, 변수, 메서드)만으로 만든 객체입니다.

- ##### Annotation 과 notation
	- 어노테이션(Annotation)과 노테이션(Notation)은 ==모두 '표기'라는 공통점==이 있지만, 사용 맥락과 방식에서 뚜렷한 차이가 있습니다. 
		- 어노테이션(Annotation): 주로 문서, 코드, 자료 등에 주석이나 설명을 덧붙여 이해를 돕는 표기법을 의미합니다. 예를 들어, 자바 프로그래밍에서 어노테이션은 @Override, @Deprecated 등과 같이 메타데이터를 전달하는 역할을 함
		- 노테이션(Notation): 수학, 음악, 과학 등에서 기호나 표기법을 사용해 정보를 기록하는 방식을 뜻합니다. 
		- 즉, 노테이션은 정보를 표기하는 일반적인 규칙이고, 어노테이션은 그 규칙 중 하나로, 프로그램에게 특정 기능을 지시하기 위해 코드에 붙이는 @표식이라고 이해하시면 됩니다.
	    - 쉽게 말해 annotation은 @ 붙은 것이자 notation을 강조한 것

            - Bean (`@Bean`): 스프링 프레임워크가 생성하고 관리하는 객체를 '빈(Bean)'이라고 부릅니다. ◦ `@Component`나 `@Bean`을 붙이면 내가 만든 POJO 객체를 스프링이 대신 관리하고 필요한 곳에 넣어줍니다. (제어의 역전, Inversion of Control)
    - DTO와 Map 구조
        - Map (Key-Value): 딕셔너리와 같은 키와 값의 쌍으로 이루어진 비선형 구조입니다.
        - DTO (Data Transfer Object): 데이터 전송을 위한 객체입니다. 필기에서는 "DTO도 Map 구조다"라고 했는데, 이는 DTO가 데이터를 이름(필드명)과 값으로 저장한다는 점에서 Map과 역할이 유사함을 강조하는 것으로 볼 수 있습니다. (실제로는 DTO는 Map이 아닌 클래스로 구현됩니다.)
        - 스키마 (Schema): 데이터베이스에서 테이블의 구조, 즉 컬럼의 집합을 의미합니다. DTO의 필드들이 보통 이 스키마의 컬럼과 일치합니다.
