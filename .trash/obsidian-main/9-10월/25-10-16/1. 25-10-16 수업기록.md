- 이름을 불러줘야 의미를 가짐. [ Scanner scanner = new Scanner ] 에서 scanner라는 이름을 지정하기 전까지는 실존하지 않음(디스크상에만 존재). 타입을 주고 이름을 주는 순간 메모리 주소로 할당되고 존재하게 됨.
- 메모리가 디스크로 내려가는 게 세이브, 디스크에서 메모리로 올라가는 게 로딩
- 신이 디스크이고, 인간은 신을 본 따 만든 형상이다(메모리이다)
- -----------
- 디스크(인간세계에 존재하는 개념)에 있는 객체(속성+기능)을 메모리에 명명한다. →instants(객체이름) 메모리(기계세계)에 개념을 주입하는 과정
- primitive type(**int**)은 subject(기계)의 입장에서 자신의 개념하에 존재하는 것 - 이미 존재하기 때문에 instants가 필요없음 → variable
- reference type(**String**)은 subject(기계)의 입장에서 사람의 개념을 참조하여 만든 개념 → 명명이 필요함
	- 자바 타입 두가지
		primitive type  
		reference type  (프리미티브를 참조)
- 즉 사람의 개념을 참조하여 만들었기 때문에 ‘기계인 나’가 아니기 때문에 객체임 명명(instants에 개념주입)

- 인스턴스(instants): 설계도와 같은 '클래스'에 기반하여 메모리를 할당받고 구체적인 데이터와 동작을 가진 실체인 '인스턴스'를 생성 / 메모리 입장에서 실체. => 객체의 이름. 인스턴스(1회용, 햄버거 한번 먹고 치움) , 실제로 한번 쓰고 치우니까 가비지(쓰레기)가 되됨
    
    - 자바에서 **인스턴스(Instance)**는 쉽게 말해 **클래스(Class)라는 설계도를 바탕으로 실제로 만들어낸 구체적인 실체 (객체)**입니다.
    - **메모리 할당 (new 키워드) 🧱**
    
    인스턴스는 클래스를 `new` 키워드와 함께 사용할 때 비로소 생성되어 컴퓨터 메모리(**힙 영역**)에 저장됩니다.
    
    ```
      {클래스 변수 이름} = \\text{new 클래스 이름();}$$
      이 `new` 작업이 바로 "붕어빵 틀(클래스)을 가지고 실제 붕어빵(인스턴스)을 찍어내는" 과정이며, 이 과정 후에만 인스턴스를 사용할 수 있게 됩니다.
    ```

- 알고리즘: 문제해결을 위한 명령들의 집합
    - 명령(boolean, instraction)
    - 하나의 행동(action)을 수행하도록 -> form 태그에 나와있는 action도 같은 맥락.
- 명령은 컴퓨터가 이해할 수 있고 실행할 수 있는 최소한의 동작 단위
    
- **Statement**는 “프로그래머가 작성한 논리적 명령 단위” 내입장
    
- **Instruction**은 “CPU가 실제로 수행하는 물리적 명령 단위” 기계 입장
    
- 메모리에서는 인스트럭션, 디스크에서는 메소드
    
    - 메소드 : 블록으로 둘러쌓인 Expression과 statement의 집합(기능)
- cpu는 계산기와 같은 것이다. 5+2를 저장하지 않고 연산만 함. cpu는 하나만 기억할 수 있음
