- 리액트 - 스프링 - 파이썬 - DB
	- 모든 것은 속성과 기능이 있고, 어느쪽에 더 포커싱 하는지가 있는 것 뿐
	- React (프론트엔드) -> 속성
	- Spring -> 기능
	- Python(백엔드) -> 기능
	- DB(데이터베이스) -> 속성
		- 위 네가지로 구성된 웹 애플리케이션 시스템을 통틀어 기술 스택(Technology Stack) 또는 풀 스택(Full Stack) 라고 함
		- 네 개는 각각의 타입을 갖고 있음. 하나로 연결되려면 타입이 같아야겠지
	- 리액트를 제외한 세 가지는 도커를 사용해야 함.
	- 스프링-자바, 파이썬-파이썬, ==DB-SQL(속성)==로 서로 다른 언어를 사용하기 때문에 연결해야 함
- 도커의 타입은 이미지
	- 도커 이미지에서 이미지란 class 즉, type 임
	- type은 객체이므로 속성과 기능을 갖고 있음
	- ==도커 이미지 풀에서 pull은 자바에서 import 하는 것과 똑같음(퀵 픽스처럼 자동으로 되지 않음. 수동으로 해야 함)==
- 컨테이너의 타입은 인스턴스
	- 도커에서 컨테이너란 도커 이미지를 실행할 수 있는 이미지를 의미함
	- 도커 컨테이너에 대해 실행, 중지, 재실행, 삭제 등의 명령을 내릴 수 있음 (= CRUD)
	- -a 라는 옵션을 주면 모든 컨테이너의 리스트를 불러오라는 뜻
	- -it 옵션
- ==도커에서는 뉴 안해도 돼. 자동으로 해서 run을 해줌== (docker container run)
- 이미지는 추상화 된 것, 접속은 컨테이너만 할 수 있음

- 테이블 = 엑셀에서 시트 한 장

| **약자** | **동작 (한글)** | **데이터베이스 작업 (SQL)** | **비유 (노트)**           |
| ------ | ----------- | ------------------- | --------------------- |
| C      | Create (생성) | `INSERT`            | 새 노트를 작성하고 저장합니다.     |
| R      | Read (조회)   | `SELECT`            | 저장된 노트를 펼쳐서 읽어봅니다.    |
| U      | Update (수정) | `UPDATE`            | 노트의 내용을 고쳐서 다시 저장합니다. |
| D      | Delete (삭제) | `DELETE`            | 다 쓴 노트를 찢어버리거나 버립니다.  |

| SQL(데이터베이스 언어)      | insert      | update     | delete        | select     |
| ------------------- | ----------- | ---------- | ------------- | ---------- |
| CRUD                | Create      | Update     | Delete        | Read       |
| java                | save        | update     | delete        | find       |
| Spring annotation @ | Postmapping | Putmapping | Deletemapping | Getmapping |
