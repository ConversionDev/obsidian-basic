- ##### JPA 구조
	- ==JPA==는 Java Persistence API의 줄임말로, 자바 프로그램에서 데이터베이스(DB)를 더 쉽고 객체 지향적으로 다루기 위한 표준입니다.
	- 쉽게 말해, ==자바 객체(클래스)와 DB의 테이블을 연결해주는 기술==이라고 생각하면 됩니다. 이 과정을 ORM(Object-Relational Mapping, 객체 관계 매핑)이라고 부릅니다.
	- 자바는 객체 지향 프로그래밍을 하고, DB는 관계형 데이터베이스로 데이터를 저장해요. 이 둘은 데이터를 다루는 방식이 달라서(패러다임의 불일치), 데이터를 저장하거나 가져올 때마다 개발자가 복잡한 SQL 문(데이터베이스 언어)을 직접 작성해야 했습니다.
	- JPA는 이 복잡한 SQL 작성과 객체-DB 간의 변환 작업을 대신 처리해주는 다리 역할을 합니다.
		-  Entity (엔티티) 
			- 역할: 데이터베이스의 테이블과 1:1로 매핑되는 자바 클래스예요.
			- 특징: 클래스 위에 `@Entity`라는 특별한 표시(어노테이션)를 붙여서 "너는 DB의 테이블이야"라고 알려줍니다. 엔티티 객체의 속성(필드) 하나하나가 테이블의 컬럼이 됩니다.
		- EntityManager (엔티티 매니저)
			- 역할: 엔티티(객체)를 관리하는 관리자이자, DB 작업을 수행하는 인터페이스예요.
			- 특징: 개발자가 `save()`, `find()`, `remove()` 같은 메서드를 사용해서 객체를 저장, 조회, 삭제할 때, 이 엔티티 매니저에게 명령합니다. 엔티티 매니저는 이 명령을 받아서 내부적으로 적절한 SQL을 생성하고 DB와 통신합니다.
		- Repository (리포지토리)
			- 역할: 엔티티 매니저를 활용해서 DB에 접근하는 코드를 더 깔끔하게 만들어주는 계층이에요 (주로 Spring Data JPA에서 사용).
			- 특징: 개발자는 여기에 `findById(id)`와 같은 메서드를 정의만 해두면, Spring Data JPA가 알아서 그 기능을 구현해줍니다. 개발자는 복잡한 DB 접근 코드를 작성할 필요 없이 메서드 호출만으로 DB 작업을 완료할 수 있어요.
