- 자료구조(Data Structure)는 데이터를 효율적으로 저장하고 관리하는 방법으로 선형과 비선형으로 나뉨
	- linear(선형구조)=list(index + value)
	- non-linea(비선형구조)=ditionary(keyword + value)
    - 선형(Linear): 리스트, 배열(array)처럼 데이터를 순서대로 저장하고, 데이터를 찾을 때는 목차(index) 즉, 숫자 순서를 사용
    - 비선형(Non-Linear): 딕셔너리(Dictionary), 맵(map)처럼 데이터를 저장할 때 이름표(key)를 붙여서 저장하며 찾을 때도 key를 사용함
        - 파이썬: list + dictionary / 자바: array + list + map
        - 검색할 때 차이점
            - **리스트(선형):** "3번째에 있는 데이터를 줘" ($\rightarrow$ 인덱스로 찾음)
            - **딕셔너리(비선형):** "이름이 'gildong'인 데이터를 줘" ($\rightarrow$ 키로 찾음)
        - Dictionary는 key 와 value의 집합. List는 index와 value의 집합
        - list 안에 map이 있고, list 안에 있을 때 element, 밖에 있을 대 map 임
- 레파지토리는 엔티티(개체)니까 기능을 빼야 함
- Data와 Data Structure의 차이: data는 단수, data structure은 복수
- 구조화된 데이터는 데이터들로 복수이며, 0개도 데이터베이스임. 빈칸을 만들었다는 뜻

- 최신 자바 웹 서비스는 Spring Boot라는 프레임워크를 기반으로 하며, 데이터 처리는 JPA(전자정부표준 데이터베이스 프레임워크)를 통해 효율적으로 이루어짐
    
    - JPA (Java Persistence API)
        - **역할:** 자바의 **객체 지향 프로그래밍(OOP)** 기술을 사용하여 데이터베이스를 다룰 수 있게 해주는 표준
        - **핵심:** 개발자가 복잡한 **SQL 쿼리**를 직접 작성하는 대신, 자바 객체(Entity)를 마치 데이터베이스 테이블처럼 다룰 수 있게 해줍니다.
- 파이프라인
    
    - 타임리프(화면) ↔ 컨트롤러 ↔ 서비스 ↔ 레파지토리 ↔ 데이터베이스
        - **비즈니스 로직 (서비스):** 데이터 조작, 계산 등 핵심적인 **알고리즘(식과 문의 조합)**이 들어가는 영역입니다. (예: 할인율 계산, 재고 확인 등)
        - **레포지토리 (Repository):** DB와 통신을 담당하는 저장소입니다. 여기서 엔티티(Entity, 개체)를 통해 DB 데이터를 자바 객체로 변환합니다.
- 의존성 주입 (Dependency Injection, DI)
    
    - **의미:** 어떤 객체가 필요로 하는 기능(의존성)을 외부(스프링 컨테이너)에서 만들어서 **주입**해 주는 기술입니다.
    - **비유:** '스파이더맨에게 거미의 능력을 집어넣는 것'처럼, 필요한 기능을 직접 만들지 않고 시스템이 제공해 주는 능력을 사용하게 됩니다. (예: `csv를 사용할 수 있는 능력`을 주입받아 사용)
