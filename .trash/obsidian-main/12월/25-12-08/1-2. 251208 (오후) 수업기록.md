- 오전: 척도 전까지. 척도는 피처에 대한 내용. 오후는 척도 이후 내용

- 모든 데이터의 컬럼은 다음 네 가지 형태 중 하나이다.
	- Nominal : 이름만 구분(문자냐 숫자냐 의미 없음), 순서 없음, 우위가 없음
	- Ordinal : 순서 있음, 우위가 있음, 간격 일정하지 않음
	- Interval: 간격 일정, 절대적 0 없음
	- Ratio : 절대적 0 있음, 모든 연산 가능 (2배, 3배 의미 있음)

- PassengerId - Ordinal
- Pclass - Ordinal
- Name - Nominal
- Sex - Nominal
- Age - Ratio
- SibSp - Ratio
- Parch - Ratio
- Ticket - Nominal
- Fare - Ratio
- Cabin - Nominal
- Embarked - Nominal

- PassengerId 와 Survived 는 제외하고 나머지 컬럼들로 데이터 컬럼을 4가지로 분류
- SibSp, Parch, Ticket, Cabin 4가지는 생존여부에 거의 관계가 없대서 메서드 코드에서 안 만듦
- title 로 묶어 : Name 정보에서 Mr, Mrs, Miss, Master --> 이런 부분들이 생존여부에 영향을 많이 준대. 그래서 따로 뺐음
- bins = [-1, 0, 5, 12, 18, 24, 35, 60, np.inf] -1은 무임승차로 인해 승객정보가 없어서 나이를 모르는 경우임. null 처리

- 정수와 실수
	- 정수면 Nominal 이거나 Ordinal
	- 실수면 Interval 이거나 Ratio
