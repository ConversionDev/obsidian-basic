### 📌 주요 내용

1. **현재 상황 분석**
   - 이중 게이트웨이 구조의 문제점
   - 아키텍처 다이어그램

2. **통합 전략 개요**
   - 목표 아키텍처
   - FastAPI Gateway 제거 및 Spring Gateway로 통합

3. **단계별 실행 계획**
   - **Phase 1**: FastAPI 서비스 독립화 (Crawler, Chatbot)
   - **Phase 2**: Spring Gateway 라우팅 추가
   - **Phase 3**: Docker Compose 통합
   - **Phase 4**: 테스트 및 검증

4. **기술적 고려사항**
   - OpenAPI 스펙 차이 처리
   - CORS 설정 통합
   - 네트워크 통신
   - 모니터링 및 로깅
   - 성능 최적화

5. **마이그레이션 체크리스트**
   - 8개 Phase별 상세 체크리스트
   - 총 40개 이상의 체크 항목

6. **실행 순서 및 예상 시간**
   - 총 8-15시간 소요 예상

### 🎯 핵심 전략

```
기존: 클라이언트 → FastAPI Gateway (9000) → Crawler/Chatbot
                  → Spring Gateway (8080) → Spring Services

변경: 클라이언트 → Spring Gateway (8080) → 모든 서비스
                                          ├─ Crawler (FastAPI)
                                          ├─ Chatbot (FastAPI)
                                          └─ Spring Services
```
