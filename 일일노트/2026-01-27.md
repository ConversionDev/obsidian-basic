- 인터프리터(Interpreter) : 코드를 한 줄(또는 한 단위)씩 읽어서 즉시 실행하는 실행기
- 컴파일러 : 전체를 미리 번역
- 허브 (Hub / 중앙 노드) : 모든 주변 기기(리프)가 일대일(Point-to-Point)로 직접 연결되는 중앙 연결 장치
- 리프 (Leaf / 주변 노드) : 중앙의 허브에 연결된 개별 컴퓨터, 워크스테이션, 프린터, 서버 등 네트워크의 종단 장치(End Device)

- 선 : spoke, 중간 : 허브
- 스타 토폴리지로 구성된 이미지에서 파일 이름을 정할때 허브와 스포크로 이름을 짓는것이 명확하다.

- 
- SQL Alchemy - DML(여러줄) CRUD
- Alembic - DDL(테이블) CRUD
- `Service Worker의 역할`
- Service Worker는 브라우저와 서버 사이에서 동작하는 프록시(자기것이 아닌것들) 스크립트입니다. 웹 페이지와 별도로 백그라운드에서 실행됩니다.
- `주요 역할`
- 1. 오프라인 지원
- 네트워크 요청을 가로채서 캐시된 리소스를 제공
- 인터넷 연결이 없어도 앱 사용 가능
- 오프라인 페이지 표시 가능

- 2. 캐싱 관리
- 정적 파일(HTML, CSS, JS, 이미지)을 캐시
- 네트워크 요청을 캐시로 대체해 로딩 속도 향상
- Cache Storage API로 캐시 관리

- 3. 백그라운드 동기화
- 페이지가 닫혀도 백그라운드에서 작업 수행
- 백그라운드 동기화, 푸시 알림 등 지원

- 4. 네트워크 요청 제어
- 모든 네트워크 요청을 가로채서 처리
- 전략 예:
- NetworkFirst: 네트워크 우선, 실패 시 캐시
- CacheFirst: 캐시 우선, 없으면 네트워크
- StaleWhileRevalidate: 캐시 먼저 제공, 백그라운드에서 업데이트

- 5. PWA 핵심 기능
- PWA 설치 가능하게 함
- 홈 화면 추가 시 네이티브 앱처럼 동작
- 오프라인에서도 기본 기능 제공
