### 옵션 1: Preview Branch 사용 (Neon만)
Production: Main Branch
Development: Preview Branch (dev)
- 장점: 클라우드에서 모든 환경 관리, Git과 연동 가능
- 단점: 인터넷 필수, 비용 가능

### 옵션 2: 표준 방식 (로컬 + Neon)
Development: 로컬 PostgreSQL
Production: Neon DB Main Branch
- 장점: 개발은 빠르고 오프라인 가능
- 단점: 환경 분리 관리 필요
## 권장 사항
Neon Preview Branch를 사용하면:
- Git 브랜치와 연동 가능
- 각 기능마다 독립 DB 환경
- 프로덕션과 동일한 환경에서 테스트
- 자동화 가능
